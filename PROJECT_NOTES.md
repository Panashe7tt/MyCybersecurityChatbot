# Project Notes

## Learning Objectives

This project demonstrates:

1. C# programming fundamentals
2. WPF desktop application development
3. Object-oriented design principles
4. JSON data serialization
5. UI/UX design with XAML
6. Natural language processing simulation
7. Application architecture patterns

## Design Decisions

### Why WPF?
WPF provides rich UI capabilities and data binding features suitable for a chat application.

### Why JSON for Storage?
JSON offers human-readable data storage that is easy to debug and modify.

### Why Keyword-Based NLP?
Simpler to implement while still demonstrating NLP concepts without external libraries.

## Lessons Learned

- Separation of concerns improves maintainability
- UI logic should be separate from business logic
- JSON serialization simplifies data persistence
- Natural language processing can be simulated effectively with keywords

## Known Limitations

- Sentiment detection is keyword-based, not true NLP
- Reminder detection only supports English words
- Quiz questions are static (not loaded from external source)
- No user authentication

## Future Improvements

- Add support for loading quiz questions from JSON
- Implement true NLP with ML.NET
- Add export/import functionality
- Create user profiles
- Add more interactive features
